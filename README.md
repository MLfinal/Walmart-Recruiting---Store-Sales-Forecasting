# Walmart-Recruiting---Store-Sales-Forecasting


ფინალური პროექტი არის kaggle ის competition: Walmart Recruiting - Store Sales Forecasting

შესრულების ინსტრუქცია:

ფინალური პროექტი კეთდება 2 კაციან გუნდებში.

მოცემული ამოცანა არის Time-Series Problem - რასთან მიმართებაშიც არსებობს რამდენიმე მოდელის არქიტექტურა:

    Deep Learning - N-BEATS, Temporal Fusion Transformer, PatchTST, DLinear (TFT სა და PatchTST ის შორის ორივე არ არის აუცილებელი).
    Tree-Based Models - LightGBM, XGBoost
    Classical Statistical Time-Series Models - ARIMA, SARIMA, Prophet (ARIMA/SARIMA ძველი მოდელებია და უფრო მნიშვნელოვანია თეორიულად გარჩევა, მათ ტრენინგზე დიდი დრო არ დახარჯოთ)
    Foundation Models - TimesFM (ამ მოდელს ბონუსად ვწერ და არ არის სავალდებულო, Google - ის ახალი Pretrained მოდელია და თუ გატესტავთ საინტერესოა ჩვენს ამოცანაზე როგორი შედები ექნება).

თქვენი ამოცანაა პროექტზე მუშაობისას გამოიკვლიოთ სხვადასხვა არქიტექტურის მოდელები, მოარგოთ თქვენს ამოცანას და შეაფასოთ რომელს ჰქონდა უფრო კარგი შედეგი და რატომ. უნდა მოიძიოთ რა განსხვავებული მიდგომები არსებობს ასეთი მოდელების დასატრენინგებლად, შესაფასებლად და რამ გამოიწვია მათი საჭიროება.

იქიდან გამომდინარე, რომ ეს სფერო მაღალი ტემპით ვითარდება, ახალ ამოცანაზე მუშაობის დაწყებისას პირველი ეტაპი ყოველთვის არის პროექტთან დაკავშირებული სხვადასხვა არქიტექტურების მოკვლევა/გარჩევა, რადგან შეუძლებელია ყველანაირი მოდელი ფუნდამენტურად იცოდე და გახსოვდეს ყოველთვის. 

შესაბამისად, ფინალურ პროექტში ყურადღება მიექცევა იმას, თუ რამდენად კარგადაა უცხო არქიტექტურები გარჩეული და იმპლემენტირებული. 

ამ დავალებისათვის უნდა შექმნათ Github ის რეპოზიტორია(რეპოზიტორიის ლინკი უნდა ატვირთოთ კლასრუმზე), ხოლო ექსპერიმენტები უნდა იყოს დალოგილი MLflow, Wandb - ზე.

რეპოზიტორიაზე უნდა იყოს შემდეგი ფაილები:

    model_experiment_{მოდელის არქიტექტურა}.ipynb - ყველა მოდელისათვის ცალცალკე ასეთი ტიპის notebook ფაილი.
    model_inference.ipynb - ამ ფაილში უნდა ხდებოდეს test set ზე პროგნოზი, საუკეთესო მოდელის გამოყენებით უნდა დააგენერიროთ submissions, რომელსაც ატვირთავთ kaggle - ზე და შეაფასებთ თქვენს მოდელს. ეს მოდელი უნდა იყოს შენახული Model Registry - ში და იქიდან უნდა დაალოუდოთ პირდაპირ და ისე გამოიძახოთ predict.
    README.md - ქართულ ენაზე დეტალურად უნდა აღწეროთ ყველა ეტაპზე რა მიდგომები გატესტეთ და თითოეულს რა შედეგი ჰქონდა.


MLflow ზე ექსპერიმენტები და ლოგები უნდა იყოს შემდეგი სტრუქტურით:

    ყველა მოდელის არქიტექტურისათვის უნდა იყოს ცალკე ექსპერიმენტი. ამ ექსპერიმენტის შიგნით კი სხვადასხვა run ები უნდა შეესაბამებოდეს preprocessing ნაწილს, feature selection, cross validation და ა.შ. მაგალითად თუ ვმუშაობთ XGBoost მოდელზე model_experiment_XGBoost.ipynb ფაილში, შესაბამისად MLflow ზე უნდა შევქმნათ ექსპერიმენტი XGBoost_Training, რომლის შიგნით იქნება შესაბამისი run - ები: XGBoost_Cleaning, XGBoost_Feature_Selection და ა.შ.
    ამ დავალებაში აუცილებელია ყველა არქიტექტურისათვის საუკეთესო მოდელი იყოს შენახული Pipeline - ად. ეს pipeline უნდა ეშვებოდეს პირდაპირ test set ზე, რომელიც preprocessed არ არის. ამ მოდელებისაგან საუკეთესო შედეგის მქონე კი უნდა იყოს დარეგისტრირებული Model Registry - ში.

ნეირონული ქსელებისათვის შეგიძლია გამოიყენოთ როგორც MLflow ასევე Wandb. ეს თქვენზეა დამოკიდებული.

იქიდან გამომდინარე, რომ ფინალური პროექტი არის გუნდური:

    რეპოზიტორია უნდა იყოს ამოცანის საერთო, ექსპერიმენტები კი როგორც თქვენთვის უფრო მოსახერხებელი იქნება ისე შეინახეთ.
    პირველ ეტაპზე ერთობლივი სამუშაო იქნება Time-Series ამოცანის სპეციფიკაში გარკვევა. შემდგომ კი უნდა გადაინაწილოთ სხვადასხვა მოდელები და იმუშაოთ დამოუკიდებლად, რომ პროექტის ბოლომდე ბევრი მოდელის გატესტვა მოასწროთ. ასევე შესაძლებელია ერთი და იგივე მოდელის სხვადასხვა ჰიპერპარამეტრებით გატესტვა პარალელურად.
    EDA, Feature Engineering და ზოგადად მონაცემების დამუშავება გააკეთეთ ერთად, მოდელების ტრენინგისას კი რაც შეიძლება ხშირად გაუზიარეთ ერთმანეთს რა და როგორ გააკეთეთ.

შეფასება:

გაითვალისწინეთ, რომ პრეზენტაცია არის სავალდებულო. სლაიდები არ არის აუცილებელი, თუმცა რეკომენდირებულია რადგან პრეზენტაცია არის 10 წუთიანი და ბევრად გაგიმარტივდებათ თქვენც მოყოლა და მეც შეფასება.
